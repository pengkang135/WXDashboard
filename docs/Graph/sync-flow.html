WXDashboard 同步流程

sync_engine.py — 数据采集的完整执行链路

前置条件：微信已运行。daemon 随 sync() 按需启停 — sync 入口 start_daemon() 触发，sync 出口 (finally) stop_daemon() 强制杀死。离开 sync 后零 wx-cli 进程存活。
PHASE 0 · HTTP 触发
▶
用户点击刷新 / POST /api/sync/refresh
前端展示进度面板，创建后台线程执行 sync()
app.py → threading.Thread → sync(progress=SyncProgress)
前端触发
PHASE 1 · discovering
🔍
发现新群
调用 wx sessions 获取全部会话，匹配 PROJECT_RULES 关键词（如 laldia、港湾），过滤 SKIP_KEYWORDS（如通威），对匹配到的新群提取群主、推断分类后入库
_discover_new_groups() → _infer_project() → _infer_category() → upsert_group()
群间延迟 3-8s
PHASE 2 · syncing_new（如有新群）
⇩
新群全量同步
对每个新群通过 wx history 拉取最近 500 条消息（LOW RISK），解析后写入 messages 表，同步提取联系人信息
safe_history(name, limit=500) → _store_group_messages() → upsert + extract contacts
批量 500 条/群
PHASE 3 · syncing（核心阶段）
↻
全量增量同步 — 单次调用
单次调用 wx new-messages --json -n 500 获取跨全部会话的增量消息（LOW RISK，基于检查点）。返回的消息按群名分组，过滤项目关键词，逐群入库。图片/视频消息从 content 中正则提取 local_id 用于去重
safe_new_messages(limit=500) → 按 chat 分组 → _store_group_messages()
单次 API 调用 | 500 条上限
PHASE 4 · files
📄
下载新文件
扫描 messages 表中未被标记为已下载的文件消息，通过 wx-cli 导出文件到本地 .Download 目录
file_downloader.download_new_files()
按需下载
PHASE 5 · cleanup
✗
杀死 daemon 进程 (finally 块保证执行)
stop_daemon() → wx daemon stop 强制终止 daemon 进程。_safe_mode_available = False 重置缓存。无论正常/异常都执行，确保离开 sync 后零 wx-cli 进程存活。
try: ... finally: stop_daemon(); _safe_mode_available = False
防止封号: 强制杀进程
PHASE 6 · done
✓
同步完成 - 零 wx-cli 进程存活
合并各阶段统计（新群数、新增消息数、更新群数、下载文件数），写入 sync_log 审计日志。所有 wx-cli 进程（daemon + 子进程）已全部退出，仅 Flask 留存。前端轮询检测到 running=false 后展示结果摘要并刷新视图
stats → add_sync_log() → progress.finish() → 前端 refreshCurrentView()
