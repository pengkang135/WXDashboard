wx-daemon 没有独立的 start 子命令；首次调用任何 wx 命令时自动 attach 微信进程提取密钥。
start_daemon() 通过两次 status 调用实现: 第一次触发自动启动, 第二次验证就绪。

CLAUDE.md 五条不可协商规则 软性禁令
1. 唯一调用者: 只有 sync_engine.py 可调用 wx-cli
2. AI 绝对禁止: 主会话及子 Agent 均不得调用 wx-cli
3. 数据唯一起源: AI 读取必须来自 SQLite
4. 禁止子 Agent: 本项目不使用 Agent/子 Agent
5. 自动阻断已部署: PreToolUse hook 硬性阻断
行为准则 依赖人工遵守

单次 sync() 完整调用链 & 频率控制点

T+0.0s _check_safe_mode() → wx daemon status  无频率限制
T+0.5s _discover_new_groups() → wx sessions -n 200 --json  无频率限制
T+1.0s 新群全量: wx export <群名> -n 500  限速: _run_wx (1.5s间隔)
T+2.5s _human_delay(0.5-2.0s)  防御: 群间随机延迟
T+3.5s 已有群增量: wx export <群名> --since <日期> -n 200  限速: _run_wx (1.5s间隔)
T+N*3s 每群重复: delay + export ... 直到所有需要同步的群完成
T+结束 download_new_files() → 纯文件系统拷贝 (无 wx-cli)  安全
