L1 wx-cli Daemon (npm) — 按 需 启 停 · 同步结束必须退出
wx-cli daemon 按需启停
npm: @jackwener/wx-cli (全局安装) | Named Pipe: \\.\pipe\wx-cli-daemon
sync() 开始时通过 init_daemon() 触发 attach 微信进程提取 SQLCipher 密钥， 缓存在 daemon 进程内存。所有 wx 子进程通过 Named Pipe 连接它查询数据。 sync() 结束前必须调用 stop_daemon() 杀死 daemon 进程， 避免微信反作弊系统检测到异常进程而强制封号。
密钥仅存在于 daemon 进程 sync 结束即退出 (stop_daemon) sync 期间 attach 微信进程
封号风险: daemon 进程持续运行
已两次因 wx-cli 进程被微信强制下线。daemon 必须在 sync 完成后立即退出。
wx 子进程 (一次性)
subprocess: wx sessions / wx history / wx new-messages

一次性子进程，存活 1-2s 全部随 sync 结束而退出

sync_engine.py 同步编排器 · 按需触发
backend/sync_engine.py
sync() — 统一入口，按需执行，非持续运行:
Phase 1 发现新群 → Phase 2 新群全量 (safe_history, 逐群子进程) → Phase 3 增量同步 (safe_new_messages, 1个子进程) → Phase 4 文件下载
每阶段产生 1~N 个 wx 子进程。同步完成后返回统计结果，所有子进程退出，仅 daemon 留存。
触发方式: POST /api/sync/refresh (Flask 后台线程) 或 start_ledger.bat (独立 Python 进程)
统一入口 sync() 增量单次调用 (1 子进程) 群间延迟 3-8s 同步完成后子进程全退

CLAUDE.md 五条不可协商规则 软性禁令
1. 唯一调用者: 只有 sync_engine.py 可调用 wx-cli
2. AI 绝对禁止: 主会话及子 Agent 均不得调用 wx-cli
3. 数据唯一起源: AI 读取必须来自 SQLite
4. 禁止子 Agent: 本项目不使用 Agent/子 Agent
5. 自动阻断已部署: PreToolUse hook 硬性阻断

单次 sync() 完整调用链 & 进程生命周期 (daemon 随 sync 启停)

T+0.0s daemon start init_daemon() → ensure_daemon() → start_daemon() 触发 daemon 启动  attach 微信提取密钥
T+0.5s 子进程 spawn _discover_new_groups() → wx sessions -n 200 --json → 子进程 exit  存活 ~1s
T+1.5s 子进程 spawn 新群全量: safe_history(群名, limit=500) → 子进程 exit  LOW RISK, 每群 1 子进程, 存活 ~2s
T+2.5s _human_delay(3-8s)  防御: 群间随机延迟 (纯 sleep, 无子进程)
T+N 子进程 spawn 增量同步: safe_new_messages(limit=500) → 子进程 exit  LOW RISK, 1 子进程覆盖全群, 存活 ~2s
T+N+1s download_new_files() → 纯文件系统拷贝 (无 wx-cli)  安全
T+N+2s daemon stop stop_daemon() → wx daemon stop 杀死 daemon  _safe_mode_available = False
T+N+3s sync() 返回 → 零 wx-cli 进程存活，仅 Flask 留存  封号风险消除
高风险 可能触发微信反作弊
中风险 有一定风险
低风险 风险可控
